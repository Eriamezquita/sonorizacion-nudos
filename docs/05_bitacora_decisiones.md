# 05. Bitácora de decisiones

Este archivo debe actualizarse conforme avance el proyecto. Sirve para justificar por qué se eligió cada representación, herramienta o mapeo.

## Decisiones iniciales

### D001. Lenguaje principal: Python

**Decisión:** usar Python como lenguaje base.

**Motivo:** permite cálculo numérico, visualización, generación de audio, notebooks reproducibles y documentación clara.

### D002. Entorno sonoro complementario: Sonic Pi

**Decisión:** usar Sonic Pi para pruebas de sonido en tiempo real.

**Motivo:** permite escuchar patrones rápidamente y ajustar decisiones musicales sin construir una infraestructura compleja.

### D003. Primer objeto matemático: nudo trébol

**Decisión:** comenzar con el nudo trébol.

**Motivo:** es el nudo no trivial más simple y permite construir un flujo completo sin perderse en complejidad computacional.

### D004. Primera representación: secuencia discreta

**Decisión:** representar el primer prototipo mediante una secuencia discreta de signos/generadores.

**Motivo:** facilita mapear cada elemento a un evento sonoro.

### D005. Primeros parámetros sonoros

**Decisión:** usar frecuencia, duración e intensidad como parámetros iniciales.

**Motivo:** son parámetros acústicos básicos, fáciles de explicar y controlar.

### D006. Documentación en español

**Decisión:** mantener documentación principal en español.

**Motivo:** el proyecto se está desarrollando en contexto académico hispanohablante y se pidió bibliografía/documentación en español.
