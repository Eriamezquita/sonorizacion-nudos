# Estado del proyecto

## Diagnóstico inicial

El proyecto ha avanzado más en conversación, planeación conceptual y exploración teórica que en el repositorio. Por eso, el primer objetivo de GitHub no es mostrar un producto terminado, sino evitar que el trabajo intelectual quede perdido en chats.

La tarea inmediata es convertir el repositorio en una memoria organizada del proyecto.

## Lo que ya existe como dirección de trabajo

- El proyecto se llama **Sonorización de nudos**.
- Forma parte de un trabajo de servicio social y tesis.
- Está asociado al laboratorio de sonorización y visualización de las matemáticas.
- Busca vincular matemáticas, física, música, arte, filología y tecnología.
- Tiene una dimensión computacional, pero también conceptual, estética y pedagógica.

## Lo que falta integrar

- Marco matemático completo.
- Mapeos sonoros explícitos.
- Criterios musicales.
- Bitácora de decisiones.
- Prototipos escuchables.
- Archivos de Sonic Pi.
- Scripts de Python.
- Documentación para reproducibilidad.
- Bibliografía verificable.
- Ruta de tesis/servicio social.
- Conexión futura con invariantes de nudos, incluyendo el invariante theta.

## Prioridad inmediata

Crear una base documental suficientemente amplia para que el repositorio refleje el verdadero alcance del proyecto, aunque el código todavía sea básico.
